# Documentación de IA para el equipo

Dos guías según tu perfil. Elige la que corresponda — o empieza por la de productividad si no estás seguro.

---

## 🧑‍💼 Perfil de negocio — IA para productividad diaria

**Para:** preventa, consultoría, solutioning, recruiting, management, operaciones — cualquier rol no técnico (y también técnicos que quieran sacar más partido a Claude.ai en su día a día).

**Sin requisitos:** funciona desde el navegador, sin instalar nada, sin saber programar.

**Qué vas a conseguir:**
- Entender qué puede hacer cada herramienta (ChatGPT, Gemini, Copilot 365, Claude.ai) y cuál merece tu tiempo
- Configurar Claude.ai para que trabaje según tu rol y tus proyectos habituales
- Generar propuestas, presentaciones y documentos Word con formato real, sin copiar y pegar
- Conectar Claude.ai a Jira, Google Drive, Azure DevOps u otros servicios sin tocar código
- Montar un agente de equipo (Project) que comparta conocimiento y configuración con todos los miembros

→ **[Ir a la guía: IA para productividad](guia-ia-productividad/README.md)**

---

## 👩‍💻 Perfil técnico — GitHub Copilot CLI y Claude Code

**Para:** desarrolladores que trabajan con Power Platform, Dataverse, PCF Components, Power Pages o cualquier stack desde el terminal.

**Requiere:** acceso a terminal (PowerShell/bash), cuenta de GitHub con Copilot o cuenta de Anthropic.

**Qué vas a conseguir:**
- Usar un agente de IA directamente desde el terminal para scaffolding, refactoring y debugging
- Configurar custom instructions, skills y agents para tu stack habitual (Power Platform, Dataverse)
- Instalar y configurar plugins y MCPs (Azure DevOps, Dataverse, GitHub Actions)
- Entender cuándo usar Copilot CLI y cuándo usar Claude Code

→ **[Ir a la guía: GitHub Copilot CLI y Claude Code](guia-developers/README.md)**
